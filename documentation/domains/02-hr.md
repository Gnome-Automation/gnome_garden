# HR Domain

**CSIA Area:** Human Resources
**Module:** `GnomeHub.HR`
**Purpose:** Team management, skills, capacity planning

---

## Overview

The HR domain manages team members, their skills, certifications, and capacity. Members extend Users with employment-specific attributes like bill rates and availability.

---

## Resources

### Member
Team members (extends User).

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| id | uuid | auto | Primary key |
| employee_id | string | no | Employee number |
| department | atom | yes | Department |
| title | string | yes | Job title |
| hourly_cost | decimal | yes | Internal cost rate |
| hourly_bill | decimal | yes | Default bill rate |
| capacity_hours | decimal | yes | Weekly availability |
| start_date | date | yes | Hire date |
| user_id | uuid | yes | Linked user account |
| manager_id | uuid | no | Reports to |
| inserted_at | utc_datetime | auto | Created |
| updated_at | utc_datetime | auto | Modified |

**Department Values:**
- `:engineering` - Engineering/design
- `:field` - Field services
- `:sales` - Sales/BD
- `:admin` - Administration

**Relationships:**
- `belongs_to :user, Management.User`
- `belongs_to :manager, HR.Member`
- `has_many :direct_reports, HR.Member`
- `has_many :skills, through: [:member_skills]`
- `has_many :certifications, HR.Certification`
- `has_many :assignments, Projects.Assignment`
- `has_many :time_entries, Projects.TimeEntry`

### Skill
Competencies and expertise areas.

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| id | uuid | auto | Primary key |
| name | string | yes | Skill name |
| category | atom | yes | Skill category |
| description | string | no | Details |

**Category Values:**
- `:plc` - PLC platforms (Rockwell, Siemens, Beckhoff)
- `:hmi` - HMI/SCADA (Ignition, FactoryTalk, Wonderware)
- `:networking` - Industrial networking
- `:programming` - Languages (ST, Ladder, Python)
- `:electrical` - Electrical knowledge
- `:mechanical` - Mechanical systems
- `:soft` - Soft skills (PM, communication)

**Common Skills:**
```
PLC: Studio 5000, TwinCAT, RSLogix 500, CLICK
HMI: Ignition, FactoryTalk View, C-More
SCADA: Ignition Perspective, Wonderware
Networking: EtherNet/IP, Modbus, OPC-UA
```

### Certification
Formal certifications with expiry tracking.

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| id | uuid | auto | Primary key |
| name | string | yes | Certification name |
| issuer | string | yes | Issuing organization |
| credential_id | string | no | Certificate number |
| issued_date | date | yes | Date earned |
| expires_date | date | no | Expiration date |
| member_id | uuid | yes | Certified member |

**Common Certifications:**
```
Inductive Automation: Ignition Core, Ignition Gold
Rockwell: ControlLogix Programming
ISA: CAP, CCST
Safety: TÜV Functional Safety
```

## Capacity Planning

```
Member.capacity_hours = 40 hrs/week

Assigned to Projects:
├── Project A: 20 hrs/week
├── Project B: 15 hrs/week
└── Available: 5 hrs/week

Utilization = (35/40) × 100 = 87.5%
```

## Key Queries

### Available Members
```elixir
# Members with capacity
HR.Member
|> Ash.Query.filter(capacity_hours > assigned_hours)
|> Ash.read!()
```

### By Skill
```elixir
# Members with Ignition skill
HR.Member
|> Ash.Query.filter(skills.name == "Ignition")
|> Ash.read!()
```

### Expiring Certifications
```elixir
# Certs expiring in 90 days
HR.Certification
|> Ash.Query.filter(expires_date < from_now(90, :day))
|> Ash.read!()
```
